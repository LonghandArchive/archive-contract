# Contributing

The Longhand Archive is a personal project, built in public, undertaken in a private capacity by [Andrew Allen](https://github.com/longhandarchive). This document describes how the archive contract engages with external interest.

## What this repository is for

The `archive-contract` repository contains the versioned specification of what collectors in The Longhand Archive produce and what consumers can rely on finding. It is the boundary between capture and derivation, documented in a form that both sides can reference.

The repository is available under [Creative Commons Attribution 4.0 International](./LICENSE). Read, quote, adapt, and reuse the specification with attribution. If the patterns described here are useful for other archival or specification work, they are free to adopt.

## What is not currently accepted

External contribution to the contract is not currently accepted. Pull requests opened against this repository will be closed without review.

Two reasons underpin this. First, the contract is an authored specification that reflects a single coherent line of architectural thinking. Accepting contributions that change or extend the specification requires a governance model — contributor agreements, versioning discipline, change review — that the project does not yet have. Second, the contract is currently being drafted against exactly one concrete collector and one concrete consumer. Its abstractions are not yet proven at the scale that would make external contribution meaningful.

Feature requests, issue reports on public repositories, and unsolicited proposals for contract changes are equivalent to pull requests for this purpose and will not be actioned.

## What is welcome

Thoughtful discussion is welcome through the channels the project maintains.

**GitHub Discussions** on the [`.github` repository](https://github.com/longhandarchive/.github/discussions) is the primary place for public conversation about the project, including observations, challenges, or questions about the archive contract.

**x.com/longhandarchive** is used for project updates and shorter-form engagement.

If you are considering building a similar specification for a different kind of archive, or adapting this contract for a different source of public data, the posture above does not apply — the work is yours, and the licence permits whatever reuse is useful.

## If the posture changes

A specification that governs multiple collectors and multiple consumers may, over time, have a stronger case for external contribution than a personal project's governance documents. If and when the posture on this repository changes, this document will be updated and the change captured as an ADR in the appropriate repository.

Until then, the posture above is the current rule.
